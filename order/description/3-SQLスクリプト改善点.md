# SQLスクリプト改善点

## 発見された問題

### 1. マジックナンバー（ハードコードされた値）

#### 1.1 ベクトル次元数1536の重複 ✅ 改善対象
- **場所**: 
  - `setup/init-scripts/02-tables.sql`
  - `setup/init-scripts/05-vector-migration.sql`
  - `setup/init-scripts/07-add-embeddings.sql`
- **問題**: ベクトル次元数が複数箇所にハードコードされている
- **影響**: 次元数を変更する際に複数箇所を修正する必要がある
- **解決策**: 定数として定義するか、コメントで一元管理を明示

### 2. コメントの問題

#### 2.1 コードから明らかなコメント
- **場所**: 複数箇所
- **問題**: 
  - `setup/init-scripts/02-tables.sql`: "商品マスタテーブル"（CREATE TABLEから明らか）
  - `setup/init-scripts/03-indexes.sql`: "インデックス作成"（CREATE INDEXから明らか）
- **影響**: コメントノイズが増える
- **解決策**: 削除または、なぜその処理が必要かを説明するコメントに変更

#### 2.2 作業内容のコメント（TODO的なコメント）
- **場所**: 
  - `setup/init-scripts/01-init.sql`: "このファイルはコンテナ初回起動時に自動実行されます"
  - `setup/init-scripts/05-vector-migration.sql`: "【注意】このスクリプトは既存データベース用です"
- **問題**: 作業内容を説明するコメント（コーディング規則違反）
- **影響**: コメントノイズ
- **解決策**: 削除または、なぜその処理が必要かを説明するコメントに変更

#### 2.3 不足している説明コメント
- **場所**: 
  - `setup/init-scripts/03-indexes.sql`: HNSWインデックスのパラメータ（m = 16, ef_construction = 64）の説明が不足
  - `setup/init-scripts/02-tables.sql`: CHECK制約の理由が不明確
- **問題**: なぜその設定値なのか、なぜその制約が必要なのかの説明が不足
- **影響**: コードの意図が理解しにくい
- **解決策**: 理由を説明するコメントを追加

### 3. 未使用コード

#### 3.1 コメントアウトされたコード
- **場所**: `setup/init-scripts/06-vector-search-examples.sql`（全体がコメントアウト）
- **問題**: 参考用として残されているが、実際には使用されていない
- **影響**: ファイルが増えるが、実際の処理には影響しない
- **解決策**: 参考用であることを明示するか、別のドキュメントに移動

#### 3.2 実装されていない機能
- **場所**: `setup/init-scripts/07-add-embeddings.sql`
- **問題**: 実際のベクトル生成を行わず、警告のみを出力
- **影響**: ファイル名と内容が一致しない
- **解決策**: ファイル名を変更するか、実際のベクトル生成処理を追加

### 4. 一貫性の問題

#### 4.1 コメントスタイルの不統一
- **場所**: 複数ファイル
- **問題**: 
  - `setup/init-scripts/01-init.sql`: "-- 初期化スクリプトの例"（説明的）
  - `setup/init-scripts/02-tables.sql`: "-- ============================================"（区切り線）
  - `setup/init-scripts/05-vector-migration.sql`: "-- 【注意】"（警告形式）
- **影響**: 可読性が低下
- **解決策**: コメントスタイルを統一

#### 4.2 命名規則の不統一
- **場所**: インデックス名
- **問題**: `idx_products_*`と`idx_product_stocks_*`で命名規則が異なる
- **影響**: 一貫性が欠ける
- **解決策**: 命名規則を統一（実際には問題ないが、一貫性のため）

### 5. エラーハンドリングの不足

#### 5.1 エラーチェックの不足
- **場所**: `setup/init-scripts/04-seed-data.sql`
- **問題**: INSERT文でエラーが発生した場合の処理が不明確
- **影響**: 部分的なデータ投入が発生する可能性
- **解決策**: トランザクション管理の明示（既にON CONFLICTで対応済み）

### 6. 重複コード

#### 6.1 コメントの重複
- **場所**: 
  - `setup/init-scripts/02-tables.sql` と `setup/init-scripts/05-vector-migration.sql`
- **問題**: embeddingカラムのコメントが重複
- **影響**: 保守性が低下
- **解決策**: 一元管理（ただし、SQLでは制約があるため、現状維持も可）
